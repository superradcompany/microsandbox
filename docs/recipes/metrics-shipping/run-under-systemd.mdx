<Note>
This recipe runs [`msb-metrics`](/observability/msb-metrics) as a
managed system service. See the
[Deep dive](/observability/deep-dive) page for flag details and
deployment constraints.
</Note>

Drop this unit at `/etc/systemd/system/msb-metrics.service`:

```ini
[Unit]
Description=microsandbox metrics collector
After=network-online.target

[Service]
Type=simple
User=msb                                       # same user that runs msb
Environment=MSB_HOME=/var/lib/microsandbox
ExecStart=/usr/local/bin/msb-metrics otel \
          --endpoint=http://localhost:4317
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

Enable and start it:

```sh
sudo systemctl daemon-reload
sudo systemctl enable --now msb-metrics
```

Systemd sends SIGTERM on `systemctl stop`; the collector drains
buffers, calls each exporter's `shutdown()`, and exits cleanly.

## Notes

- **`User=`** must match the user that runs the `msb` runtime, otherwise
  `EACCES` on shm attach. See
  [Deep dive → Deployment constraints](/observability/deep-dive#deployment-constraints).
- **`Environment=MSB_HOME=`** must match the runtime's `$MSB_HOME`, or
  set `--msb-home=` explicitly in `ExecStart`.
- The `--endpoint=` in this example targets a local Alloy or
  otel-collector on `4317`. For a direct provider endpoint, see the
  per-backend recipes in this section.
