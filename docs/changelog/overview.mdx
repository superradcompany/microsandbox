This changelog highlights notable changes to microsandbox from early February through early May 2026, grouped by week. Each entry links the merged pull request on GitHub.

  - **Snapshot docs unified across SDKs** — Aligned language order and filled in missing examples. ([#658](https://github.com/superradcompany/microsandbox/pull/658))
</Update>

<Update label="Apr 27 – May 3, 2026" tags={["release", "v0.4.4", "v0.4.3", "agentd", "snapshot", "logs", "cli", "networking"]}>
  ## Week of April 27

  ### Releases

  - **v0.4.4** — Routine release. ([#655](https://github.com/superradcompany/microsandbox/pull/655))

  ### Features

  - **Guest init handoff** — `agentd` now hands off PID 1 to a guest init binary, improving signal handling and child reaping. ([#653](https://github.com/superradcompany/microsandbox/pull/653))
  - **File-first disk snapshots** — Unified snapshot API across CLI, Rust, TypeScript, and Python. ([#652](https://github.com/superradcompany/microsandbox/pull/652))
  - **Exec log capture** — Adds `exec.log` capture, `boot-error.json`, and a typed `ExecFailed` error. ([#650](https://github.com/superradcompany/microsandbox/pull/650))
  - **Inline `--script`** — `msb` now accepts an inline `--script` flag; the file form is renamed to `--script-path`. ([#643](https://github.com/superradcompany/microsandbox/pull/643))
  - **`msb` alias** — The packaging step installs an `msb` alias alongside the `microsandbox` CLI. ([#642](https://github.com/superradcompany/microsandbox/pull/642))

  ### Fixes

  - **`--no-net` air-gaps the guest** — Disabling networking now fully isolates the guest. ([#647](https://github.com/superradcompany/microsandbox/pull/647))
  - **Documentation** — MDX parse error fixed in `sandboxes/customization`, favicon corrected, and style consistency tightened across pages. ([#659](https://github.com/superradcompany/microsandbox/pull/659), [#657](https://github.com/superradcompany/microsandbox/pull/657), [#660](https://github.com/superradcompany/microsandbox/pull/660))
</Update>

<Update label="Apr 20 – Apr 26, 2026" tags={["release", "v0.4.0", "v0.4.1", "v0.4.2", "v0.4.3", "networking", "sdk-node", "volumes"]}>
  ## Week of April 20

  ### Releases

  - **v0.4.0**, **v0.4.1**, **v0.4.2**, **v0.4.3** — Major networking redesign and TypeScript SDK rewrite. ([#627](https://github.com/superradcompany/microsandbox/pull/627), [#638](https://github.com/superradcompany/microsandbox/pull/638), [#640](https://github.com/superradcompany/microsandbox/pull/640))

  ### Breaking changes

  - **TypeScript SDK redesign** — Builders are now native NAPI classes; the SDK ships with a builder API and bundled binaries. ([#623](https://github.com/superradcompany/microsandbox/pull/623), [#634](https://github.com/superradcompany/microsandbox/pull/634))

  ### Features

  - **Domain-rule enforcement** — Domain rules are now enforced consistently at the DNS, SNI, and policy layers. ([#633](https://github.com/superradcompany/microsandbox/pull/633))
  - **Network policy redesign** — Per-rule direction in the policy schema, new `--net-rule` grammar, and policy + DNS builders with a lazy-parse error model. ([#618](https://github.com/superradcompany/microsandbox/pull/618), [#619](https://github.com/superradcompany/microsandbox/pull/619), [#620](https://github.com/superradcompany/microsandbox/pull/620))
  - **Disk-image volume mounts** — Volumes can now be backed by disk images, with unified read-only handling. ([#609](https://github.com/superradcompany/microsandbox/pull/609))
  - **`host.microsandbox.internal` alias** — New DNS alias for reaching the host from inside a sandbox. ([#602](https://github.com/superradcompany/microsandbox/pull/602))
  - **Per-sandbox `libkrunfw_path`** — SDK now exposes a per-sandbox override. ([#599](https://github.com/superradcompany/microsandbox/pull/599))

  ### Fixes

  - **Domain & domain-suffix rules** — Improved enforcement. ([#605](https://github.com/superradcompany/microsandbox/pull/605))
  - **Installer** — Reject unsupported glibc versions. ([#635](https://github.com/superradcompany/microsandbox/pull/635))
  - **Node SDK packaging** — Ship `libkrunfw` under its canonical name in the npm platform package. ([#648](https://github.com/superradcompany/microsandbox/pull/648))

  ### Docs

  - **Networking docs aligned with new policy** — Adds the TypeScript builder. ([#616](https://github.com/superradcompany/microsandbox/pull/616))
</Update>

<Update label="Apr 13 – Apr 19, 2026" tags={["dns", "tls", "image", "agentd", "networking"]}>
  ## Week of April 13

  ### Breaking changes

  - **Block-backed OCI rootfs** — Container rootfs is now backed by a VMDK with EROFS `fsmeta`. ([#548](https://github.com/superradcompany/microsandbox/pull/548))

  ### Features

  - **DNS over TCP/53 and DoT/853** — `agentd` intercepts both. ([#585](https://github.com/superradcompany/microsandbox/pull/585))
  - **Trust host CAs inside sandbox guests** — Opt-in trust of the host CA bundle. ([#583](https://github.com/superradcompany/microsandbox/pull/583))
  - **Custom TLS certs and insecure registries** — Image pulls now support custom TLS certs and insecure registries. ([#496](https://github.com/superradcompany/microsandbox/pull/496))
  - **Secret injection in TS SDK** — TypeScript SDK gains the same secret injection options as the Rust SDK. ([#586](https://github.com/superradcompany/microsandbox/pull/586))
  - **Sandbox-wide guest rlimits** — Set rlimits across the guest. ([#549](https://github.com/superradcompany/microsandbox/pull/549))
  - **`AgentdConfig`** — `agentd` now reads env vars once at startup via `AgentdConfig`. ([#507](https://github.com/superradcompany/microsandbox/pull/507))
  - **Keyring feature flag** — Adds an opt-in keyring feature. ([#563](https://github.com/superradcompany/microsandbox/pull/563))

  ### Fixes

  - **macOS DNS resolver** — Fetch nameservers via `SCDynamicStore` on macOS. ([#582](https://github.com/superradcompany/microsandbox/pull/582))
  - **VM boot regression** — Restore VM boot and `agentd` Docker build. ([#579](https://github.com/superradcompany/microsandbox/pull/579))

  ### Docs

  - **Networking section split** — Networking docs split into overview, DNS, TLS, and security model pages. ([#584](https://github.com/superradcompany/microsandbox/pull/584))
  - **Recipes section** — New recipes section, plus updated registry, CLI, and SDK docs. ([#474](https://github.com/superradcompany/microsandbox/pull/474))
</Update>

<Update label="Mar 30 – Apr 5, 2026" tags={["release", "v0.3.8", "v0.3.9", "v0.3.10", "v0.3.11", "v0.3.12", "rewrite", "filesystem"]}>

  ### Releases

  - **v0.3.8 – v0.3.12** — Stabilization releases following the rewrite. ([#479](https://github.com/superradcompany/microsandbox/pull/479), [#489](https://github.com/superradcompany/microsandbox/pull/489), [#492](https://github.com/superradcompany/microsandbox/pull/492), [#501](https://github.com/superradcompany/microsandbox/pull/501), [#506](https://github.com/superradcompany/microsandbox/pull/506))

  ### Features

  - **File-level volume mounts** — Sandboxes can now mount individual files. ([#504](https://github.com/superradcompany/microsandbox/pull/504))

  - **`metricsStream` in TS SDK** — Attach to live metrics. ([#523](https://github.com/superradcompany/microsandbox/pull/523))

  - **Per-command exec flags** — `msb run` accepts per-command exec flags; new AI Agents section in the README. ([#485](https://github.com/superradcompany/microsandbox/pull/485))
  - **Filesystem benchmark harness** — Docker vs microsandbox harness. ([#525](https://github.com/superradcompany/microsandbox/pull/525))

  ### Refactors

  - **Reopen-on-demand filesystem** — Replaces per-inode FD pinning with a reopen-on-demand model. ([#505](https://github.com/superradcompany/microsandbox/pull/505))
  - **Simplified sandbox APIs** — `exec`/`attach`/`image` gain `_with` variants. ([#484](https://github.com/superradcompany/microsandbox/pull/484))
  - **Removed `msb shell`** — The legacy shell command was removed. ([#509](https://github.com/superradcompany/microsandbox/pull/509))

  - **Self-update** — Download the latest version bundle during `msb self update`. ([#490](https://github.com/superradcompany/microsandbox/pull/490))
  - **Symlink xattr ops** — Fix symlink xattr ops and initialize `listxattr`. ([#493](https://github.com/superradcompany/microsandbox/pull/493))

  ### Docs

  - **Navigation restructure** — SDK reference is split by language. ([#535](https://github.com/superradcompany/microsandbox/pull/535))
  - **Global `config.json` reference** — New page documenting global config. ([#511](https://github.com/superradcompany/microsandbox/pull/511))
