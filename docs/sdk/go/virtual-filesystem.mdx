Serve a guest directory from your own code instead of host storage. You implement `vfs.PathFs`, attach it with `WithVirtualMount`, and the SDK runs the provider loop for you. See [Virtual filesystem](/sandboxes/virtual-filesystem) for concepts and usage patterns.

Virtual mounts are available in the **Go** and **Rust** SDKs today. Python and Node SDKs do not expose them yet.

Builder options live in `microsandbox`; the provider interface, types, and helpers live in `github.com/superradcompany/microsandbox/sdk/go/vfs`.

<p className="msb-label" id="typical-flow">Typical flow</p>

```go
import (
    m "github.com/superradcompany/microsandbox/sdk/go"
    "github.com/superradcompany/microsandbox/sdk/go/vfs"
)

type myFs struct{ vfs.ReadOnly }

func (myFs) GetAttr(path []byte) (vfs.Attr, error) { /* ... */ }
func (myFs) ReadDir(path []byte) ([]vfs.DirEntry, error) { /* ... */ }
func (myFs) Read(path []byte, off uint64, n uint32) ([]byte, error) { /* ... */ }

sb, err := m.CreateSandbox(ctx, "worker",
    m.WithImage("alpine"),
    m.WithVirtualMount("/inbox", myFs{},
        m.WithVirtualMountCachePolicy(m.VfsCacheAuto),
    ),
)
defer sb.Close() // keep open for the sandbox lifetime; closing stops the provider
```

The SDK runs `vfs.Serve` on a goroutine per mount — you normally never call it directly. Providers must be concurrency-safe and report failures with `vfs.Err`. Virtual mounts require an attached sandbox (`WithDetached` is incompatible).

### Lifecycle

Provider sockets are tied to **VM lifetime**, not handle refcount: they stay open while the guest may perform I/O and are torn down once (idempotently) after the VM reaches stopped state. `Stop` and `Kill` on a live `Sandbox` or `SandboxHandle` tear providers down synchronously after stopped state is observed. `RequestStop`, `RequestKill`, and `Close` on a lifecycle-owning `Sandbox` schedule the same teardown in the background after the VM stops. `SandboxHandle.RequestStop` and `RequestKill` schedule teardown by sandbox name. `Close` on a `Connect()` handle only drops that handle's reference; sockets stay up until the VM stops. `RemoveSandbox` also tears down providers when this process still hosts them. Route diagnostics with `SetVirtualMountLogger`. Always call `Close` before letting a lifecycle-owning `Sandbox` go out of scope — if a handle is garbage-collected without `Close`, lifecycle owners schedule teardown after the VM stops. A running sandbox with virtual mounts cannot be booted again via `StartSandbox` — use `Connect` instead. **Stopped sandboxes that successfully created with virtual mounts cannot be restarted** — remove the sandbox record and call `CreateSandbox` with a fresh `WithVirtualMount` provider instead of `StartSandbox`. If create fails before it completes successfully, the record persists `attempted_virtual_mounts` without `had_virtual_mounts` and **`StartSandbox` and `Connect` are rejected** — remove the record or recreate with `WithReplace`, then create again. When `CreateSandbox` returns an error, the Go SDK closes any `vfs.Serve` goroutines it started; in-process provider shutdown inside the native `create` path covers Rust-managed mounts. Provider registry installation in Go happens only after a successful create; Rust registers servers at the end of successful create but starts provider-exit watches immediately after spawn succeeds. Both SDKs close every mount socket on the first unexpected provider exit. If a provider serve loop exits unexpectedly, the Go SDK shuts down every virtual-mount provider in that sandbox and requests sandbox stop. If deferred teardown keeps retrying for 30 minutes (for example a wedged VM), the background waiter forces provider shutdown even when the sandbox has not reached a terminal state. Virtual-mount guest paths must not overlap each other, any volume mount, or share a virtio-fs tag with any bind, named, or disk-image volume. Nested volume guest paths (without a virtual mount) remain allowed.

Deploy the Go SDK and the `msb` runtime from the same release when using virtual mounts: the inherited-socket protocol is currently **v4** and accepts **v3** on the handshake. A skew of two or more versions fails at boot. Mixed v3/v4 pairs can pass the hello but **`FsyncDir` (v4) is required for guest `fsyncdir` to refresh stale directory listings** on RPC mounts; without it, out-of-band provider changes stay invisible until a guest mutation invalidates the cache.

Plain `readdir` may show provisional inode numbers (`d_ino`) that change after a later `lookup` establishes the authoritative inode; `readdirplus` interns real inodes during listing. When two paths hash to the same provisional value within one listing, the scaffold salts the second so `d_ino` stays unique in that directory. Tools that compare listing vs stat should not rely on plain `readdir` `d_ino` until lookup.

`vfs.Serve` caches each connection's last full `ReadDir` listing for paginated replies and invalidates that cache when a mutating op completes on the connection. At most **64** distinct directory paths are cached per connection (LRU eviction); continuing pagination after eviction returns `EAGAIN`. A concurrent `ReadDir` that overlaps a write may still return the pre-write listing; do not rely on paginated `ReadDir` alone for strict read-your-writes ordering across parallel guest I/O. Cache invalidation may briefly overlap two in-flight `ReadDir` calls for the same path — providers must be concurrency-safe.

The runtime scaffold checks `access(2)` permission bits against the caller's uid and primary gid only; supplementary groups are not consulted.

## Options

<p id="options" />

Functional options for [CreateSandbox](/sdk/go/sandbox#createsandbox). Pass `WithVirtualMount` to `CreateSandbox`; pass the `WithVirtualMount*` tuning helpers as `opts` to `WithVirtualMount`.

| Option | Signature | Kind | Description |
|--------|-----------|------|-------------|
| `WithVirtualMount` | `func WithVirtualMount(guestPath string, provider vfs.PathFs, opts ...VirtualMountOption) SandboxOption` | `SandboxOption` | Mount a programmable filesystem at `guestPath`. Allocates a socket pair, runs `provider` on the parent end via `vfs.Serve`, and inherits the child end into the runtime. May be passed more than once; each `guestPath` must be unique and must not nest under another mount (for example `/data` and `/data/inbox` together is rejected). `guestPath` must be absolute, not `/`, must not end with `/`, and must not contain `:`, `;`, or `,`. |
| `WithVirtualMountEntryTimeout` | `func WithVirtualMountEntryTimeout(d time.Duration) VirtualMountOption` | `VirtualMountOption` | Directory-entry cache lifetime. Longer values keep repeated `lookup` calls off the channel. Rounded up to whole seconds; zero or negative leaves the default. |
| `WithVirtualMountAttrTimeout` | `func WithVirtualMountAttrTimeout(d time.Duration) VirtualMountOption` | `VirtualMountOption` | Attribute cache lifetime before the guest re-asks via `GetAttr`. Rounded up to whole seconds; zero or negative leaves the default. |
| `WithVirtualMountCachePolicy` | `func WithVirtualMountCachePolicy(policy VfsCachePolicy) VirtualMountOption` | `VirtualMountOption` | FUSE data-caching policy for guest reads (`VfsCacheNever`, `VfsCacheAuto`, or `VfsCacheAlways`). |
| `WithVirtualMountWriteback` | `func WithVirtualMountWriteback(enabled bool) VirtualMountOption` | `VirtualMountOption` | Enable FUSE writeback caching so the guest kernel can buffer and coalesce writes before they reach the provider's `Write`. |
| `WithVirtualMountCallTimeout` | `func WithVirtualMountCallTimeout(d time.Duration) VirtualMountOption` | `VirtualMountOption` | How long a single guest filesystem op waits for the provider before it is failed and surfaced to the guest as `EIO` (freeing the FUSE worker rather than pinning it). Rounded up to whole seconds; zero or negative leaves the default (30s). Raise it for slow backends such as cold object-store misses. |

## Functions

<p id="functions" />

| Function | Signature | Description |
|----------|-----------|-------------|
| `Serve` | `func Serve(conn io.ReadWriter, fs PathFs, opts ...ServeOption) error` | Run the request/response loop for one mount: read framed requests from `conn`, dispatch to `fs`, write replies. `WithVirtualMount` calls this for you. Up to `MaxConcurrentOps` (16) concurrent dispatches per mount. Malformed requests get `EIO`; clean EOF returns `nil`. Pass `WithErrorLog` to route recovered provider panics and non-EOF serve-loop exits. |
| `WithErrorLog` | `func WithErrorLog(fn func(format string, args ...any)) ServeOption` | Set the function `Serve` uses to report recovered provider panics and non-EOF serve-loop exit errors. Defaults to a no-op; `WithVirtualMount` passes `SetVirtualMountLogger`'s hook when set. |
| `SetVirtualMountLogger` | `func SetVirtualMountLogger(fn func(format string, args ...any))` | Route virtual-mount diagnostics (serve-loop exit, panics, handles garbage-collected without `Close`). Defaults to a no-op. |
| `Err` | `func Err(code int) error` | Wrap a Linux errno as `*Errno` for return from a `PathFs` method. Any other non-nil error becomes `EIO` in the guest. |

## Types

### vfs.PathFs

<span id="pathfs" />

Path-addressed filesystem backend. Paths are absolute `[]byte` values starting with `/`; names are raw bytes, not validated UTF-8. Report failures with `vfs.Err`. Embed `ReadOnly` for a read-only tree (implement `GetAttr`, `ReadDir`, and `Read` only). `ReadDir` names must be single path components (no `.`, `..`, or `/`); the server rejects invalid names before they reach the guest. Read/write payloads are capped at 128 KiB per RPC.

| Group | Method | Signature | Returns | ReadOnly default | Description |
|-------|--------|-----------|---------|------------------|-------------|
| Read | `GetAttr` | `(path []byte)` | `(Attr, error)` | — | Node attributes, or `ENOENT` if absent |
| Read | `ReadDir` | `(path []byte)` | `([]DirEntry, error)` | — | Directory children (excluding `.`/`..`) |
| Read | `Read` | `(path []byte, offset uint64, size uint32)` | `([]byte, error)` | — | Up to `size` bytes from `offset`; short result signals EOF |
| Read | `ReadLink` | `(path []byte)` | `([]byte, error)` | `ENOSYS` | Symlink target |
| Write | `Write` | `(path []byte, offset uint64, data []byte)` | `(int, error)` | `ENOSYS` | Bytes accepted at `offset` |
| Write | `Create` | `(path []byte, attr Attr)` | `(Attr, error)` | `ENOSYS` | Create a node per `attr.Kind` |
| Write | `Mkdir` | `(path []byte, mode uint32)` | `(Attr, error)` | `ENOSYS` | Create a directory |
| Write | `Remove` | `(path []byte)` | `error` | `ENOSYS` | Delete a node or empty directory |
| Write | `Rename` | `(from, to []byte)` | `error` | `ENOSYS` | Move `from` to `to` |
| Write | `SetAttr` | `(path []byte, attr Attr, valid SetAttrValid)` | `(Attr, error)` | `ENOSYS` | Apply selected `attr` fields |
| Write | `Symlink` | `(path, target []byte)` | `(Attr, error)` | `ENOSYS` | Create a symlink |
| Write | `Flush` | `(path []byte)` | `error` | no-op | Commit buffered writes for a file |
| Write | `Fsync` | `(path []byte, datasync bool)` | `error` | no-op | Sync file data (and metadata unless `datasync`) to stable storage |
| Xattr | `SetXattr` | `(path, name, value []byte, flags uint32)` | `error` | `ENOSYS` | Set extended attribute |
| Xattr | `GetXattr` | `(path, name []byte)` | `([]byte, error)` | `ENODATA` | Get extended attribute |
| Xattr | `ListXattr` | `(path []byte)` | `([][]byte, error)` | empty list | List xattr names |
| Xattr | `RemoveXattr` | `(path, name []byte)` | `error` | `ENOSYS` | Remove extended attribute |
| Info | `StatFs` | `()` | `(StatFs, error)` | generic unbounded volume | Filesystem statistics |

### vfs.ReadOnly

<span id="readonly" />

Embeddable base for read-only providers. Override `GetAttr`, `ReadDir`, and `Read`; all other methods use the **ReadOnly default** column in the `PathFs` table.

### vfs.RenameFlagsAware

<span id="renameflagsaware" />

Optional interface for providers that enforce Linux `renameat2` flags atomically (typically under their own lock). When your provider implements `RenameWithFlags`, the `Serve` loop calls it for `Rename` RPCs that carry flag bits; otherwise the server falls back to a best-effort `GetAttr` pre-check before `Rename`.

| Method | Signature | Description |
|--------|-----------|-------------|
| `RenameWithFlags` | `(from, to []byte, flags uint32) error` | Rename honoring `RenameNoReplace` (1) and rejecting `RenameExchange` (2) with `ENOSYS` |

Dynamic providers that mutate outside the guest should expose fresh listings after the guest calls `fsyncdir` on an open directory handle; until then, open handles keep their point-in-time snapshot and a non-zero `readdir` cookie after refresh returns `EAGAIN` until the client rewinds to offset `0`. Mutations invalidate only directory handles whose path is affected; unrelated open directory handles stay valid.

Before removing a directory, the runtime treats `ReadDir` as empty when every returned name fails guest validation. Providers must not hide real children behind invalid names — that would allow removing a non-empty directory.
