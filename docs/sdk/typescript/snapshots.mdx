<Note>
  VM snapshots (live capture / fork / restore) are coming soon and not yet available in the current SDK. The API below covers the related shipping feature: **rootfs patches**, which let you modify the guest filesystem before the VM boots.
</Note>

See [Snapshots](/sandboxes/snapshots) for usage examples and patterns.

## Rootfs patches

Rootfs patches are pre-boot modifications applied to the guest filesystem. Use them to seed config files, drop in scripts, or patch image defaults without rebuilding the OCI image.

```typescript
import { Sandbox } from "microsandbox";

await using sandbox = await Sandbox.builder("patched")
  .image("alpine")
  .patch((p) =>
    p
      .text("/etc/greeting.txt", "Hello!\n")
      .mkdir("/app", { mode: 0o755 })
      .text("/app/config.json", '{"debug": true}', { mode: 0o644 })
      .copyDir("./scripts", "/app/scripts")
      .append("/etc/hosts", "127.0.0.1 myapp.local\n"),
  )
  .replace()
  .create();
```

## PatchBuilder

Builder consumed by [`SandboxBuilder.patch(p => …)`](/sdk/typescript/sandbox#sandboxbuilder). Each call appends one entry to the patch list and returns the builder.

| Method | Signature | Description |
|--------|-----------|-------------|
| text | `(path: string, content: string, opts?: PatchFileOptions) => this` | Write text to a file in the guest rootfs. |
| file | `(path: string, content: Uint8Array, opts?: PatchFileOptions) => this` | Write raw bytes. |
| copyFile | `(src: string, dst: string, opts?: PatchFileOptions) => this` | Copy a host file into the guest rootfs. |
| copyDir | `(src: string, dst: string, opts?: PatchReplaceOnly) => this` | Recursively copy a host directory. |
| symlink | `(target: string, link: string, opts?: PatchReplaceOnly) => this` | Create a symlink. |
| mkdir | `(path: string, opts?: PatchModeOnly) => this` | Create a directory (idempotent). |
| remove | `(path: string) => this` | Delete a path (idempotent). |
| append | `(path: string, content: string) => this` | Append text to an existing file; copies up from a lower layer if needed. |
| build | `() => Patch[]` | Materialize the patch list. |

### Option types

```typescript
interface PatchFileOptions {
  mode?: number;     // Unix permission bits (e.g. 0o644)
  replace?: boolean; // Allow shadowing an existing path
}

interface PatchReplaceOnly { replace?: boolean; }
interface PatchModeOnly    { mode?: number; }
```

## Types

### Patch

Discriminated union representing one patch entry. Stored on `SandboxConfig.patches`.

```typescript
type Patch =
  | { kind: "text"; path: string; content: string; mode?: number; replace?: boolean }
  | { kind: "file"; path: string; content: Uint8Array; mode?: number; replace?: boolean }
  | { kind: "copyFile"; src: string; dst: string; mode?: number; replace?: boolean }
  | { kind: "copyDir"; src: string; dst: string; replace?: boolean }
  | { kind: "symlink"; target: string; link: string; replace?: boolean }
  | { kind: "mkdir"; path: string; mode?: number }
  | { kind: "remove"; path: string }
  | { kind: "append"; path: string; content: string };
```

Patch failures at boot raise [`PatchFailedError`](/sdk/typescript/sandbox#errors).
