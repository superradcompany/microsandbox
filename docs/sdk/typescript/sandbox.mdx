## Static methods

---

#### Sandbox.builder()

```typescript
static builder(name: string): SandboxBuilder
```

Begin building a new sandbox. Chain configuration setters, then call `.create()` or `.createDetached()` (see [`SandboxBuilder`](#sandboxbuilder)) to boot the VM.

**Parameters**

| Name | Type | Description |
|------|------|-------------|
| name | `string` | Sandbox name |

**Returns**

| Type | Description |
|------|-------------|
| [`SandboxBuilder`](#sandboxbuilder) | Fluent builder |

#### Sandbox.start()


| Type | Description |

Get a handle to an existing sandbox (running or stopped). The handle provides status, configuration, and lifecycle control.

#### Sandbox.list()

```typescript
static list(): Promise<Array<SandboxHandle>>
```

List all known sandboxes (running, stopped, and crashed). Each handle is read-only — call [`Sandbox.get(name)`](#sandboxget) to obtain a live handle that supports lifecycle methods.

**Returns**

| Type | Description |
|------|-------------|
| `Array<`[`SandboxHandle`](#sandboxhandle)`>` | All sandboxes |


## Instance properties

`Sandbox` instances are obtained from [`SandboxBuilder`](#sandboxbuilder)`.create()` / `.createDetached()`, [`Sandbox.start()`](#sandboxstart), [`Sandbox.startDetached()`](#sandboxstartdetached), or by calling `.start()`/`.connect()` on a [`SandboxHandle`](#sandboxhandle). Properties below are synchronous getters.

| Property | Type | Description |
|----------|------|-------------|
| name | `string` | Sandbox name. |
| config | `Readonly<`[`SandboxConfig`](#sandboxconfig)`>` | The configuration this sandbox was built from. Read-only. |
| ownsLifecycle | `boolean` | `true` for sandboxes created in attached mode (auto-stops on `Symbol.asyncDispose`); `false` for detached or reconnected sandboxes. |

## Instance methods

`Sandbox` implements `AsyncDisposable` — use `await using sandbox = ...` to auto-stop the VM at scope exit (only when `ownsLifecycle` is `true`).

```typescript
import { Sandbox } from "microsandbox";

await using sandbox = await Sandbox.builder("demo")
  .image("alpine")
  .cpus(1)
  .memory(512)
  .create();

const out = await sandbox.shell("echo hi");
console.log(out.stdout());
```

#### `[Symbol.asyncDispose]()`

```typescript
[Symbol.asyncDispose](): Promise<void>
```

Called automatically by `await using`. When `ownsLifecycle` is `true`, stops the sandbox; otherwise no-op. Errors during the dispose-stop are swallowed.

## Types

### SandboxBuilder

Fluent builder returned by [`Sandbox.builder(name)`](#sandboxbuilder-1). Every setter returns the builder for chaining. The terminal calls are `build()`, `create()`, and `createDetached()` (documented in the table below).

#### Core image / sizing

| Method | Signature | Description |
|--------|-----------|-------------|
| image | `(src: string \| RootfsSource) => this` | OCI reference (`"python:3.12"`), bind path (`"./rootfs"`), or disk image (`"./disk.qcow2"`). **Required.** |
| cpus | `(count: number) => this` | Virtual CPU count. |
| memory | `(size: Mebibytes \| number) => this` | Guest memory limit. Use [`MiB(n)`](#sizes), `KiB(n)`, `GiB(n)`, `TiB(n)` — or a bare number (interpreted as MiB). |
| logLevel | `(level: LogLevel) => this` | Override log verbosity. |
| quietLogs | `() => this` | Suppress sandbox stderr. |
| workdir | `(path: string) => this` | Default working directory for commands. |
| shell | `(shell: string) => this` | Shell binary used by `sandbox.shell(...)` (default: `/bin/sh`). |

#### Image-defined overrides

| Method | Signature | Description |
|--------|-----------|-------------|
| entrypoint | `(cmd: Iterable<string>) => this` | Override image entrypoint. |
| hostname | `(name: string) => this` | Guest hostname. |
| user | `(user: string) => this` | Default guest user. |
| pullPolicy | `(policy: PullPolicy) => this` | When to pull the image (`"always"` \| `"if-missing"` \| `"never"`). |
| libkrunfwPath | `(path: string) => this` | Override the `libkrunfw` shared library path for this sandbox. |

#### Environment, scripts, lifecycle

| Method | Signature | Description |
|--------|-----------|-------------|
| env | `(key: string, value: string) => this` | Add a single environment variable. |
| envs | `(vars: Iterable<[string,string]> \| Record<string,string>) => this` | Bulk-add environment variables. |
| script | `(name: string, content: string) => this` | Add a named script mounted at `/.msb/scripts/<name>`. |
| scripts | `(scripts: Iterable<[string,string]> \| Record<string,string>) => this` | Bulk-add scripts. |
| replace | `() => this` | Replace any existing sandbox with the same name. |
| maxDuration | `(secs: number) => this` | Maximum sandbox lifetime in seconds. |
| idleTimeout | `(secs: number) => this` | Stop the sandbox after this many seconds of inactivity. |

#### Mounts and patches

| Method | Signature | Description |
|--------|-----------|-------------|
| volume | `(guestPath: string, configure: (m: MountBuilder) => MountBuilder) => this` | Mount a volume — see [Volumes](/sdk/typescript/volumes#mountbuilder). |
| patch | `(configure: (p: PatchBuilder) => PatchBuilder) => this` | Pre-boot rootfs modifications — see [Snapshots](/sdk/typescript/snapshots). |
| addPatch | `(p: Patch) => this` | Append a single pre-built `Patch`. |

#### Registry, networking, ports, secrets

| Method | Signature | Description |
|--------|-----------|-------------|
| registry | `(configure: (r: RegistryConfigBuilder) => RegistryConfigBuilder) => this` | Configure registry auth, CA paths, and insecure mode. |
| port | `(hostPort: number, guestPort: number) => this` | Publish a TCP port `host → guest`. |
| portUdp | `(hostPort: number, guestPort: number) => this` | Publish a UDP port `host → guest`. |
| disableNetwork | `() => this` | Disable networking entirely. |
| network | `(configure: (n: NetworkBuilder) => NetworkBuilder) => this` | Full networking config — see [Networking](/sdk/typescript/networking). |
| secretEnv | `(envVar: string, value: string, allowedHost: string) => this` | Shorthand for a secret with placeholder `$MSB_<envVar>` and a single allowed host. |
| secret | `(configure: (b: SecretBuilder) => SecretBuilder) => this` | Add a fully customized secret — see [Secrets](/sdk/typescript/secrets). |

#### Terminal methods

| Method | Signature | Description |
|--------|-----------|-------------|
| build | `() => SandboxConfig` | Materialize the accumulated state without booting. Throws [`InvalidConfigError`](/sdk/typescript/sandbox) if configuration is invalid. |
| create | `() => Promise<Sandbox>` | Build and boot in attached mode (auto-stops on `Symbol.asyncDispose`). |
| createDetached | `() => Promise<Sandbox>` | Build and boot in detached mode (survives this process). |

### SandboxHandle

A lightweight handle to an existing sandbox, returned by [`Sandbox.get()`](#sandboxget) and [`Sandbox.list()`](#sandboxlist). Handles produced by `Sandbox.list()` are read-only — fetch a live handle via `Sandbox.get(name)` to call lifecycle methods.

| Property / Method | Type | Description |
|-------------------|------|-------------|
| name | `string` | Sandbox name. |
| status | [`SandboxStatus`](#sandboxstatus) | Current status. |
| configJson | `string` | Raw JSON configuration. |
| createdAt | `Date \| null` | Creation timestamp. |
| updatedAt | `Date \| null` | Last update timestamp. |
| metrics() | `Promise<`[`SandboxMetrics`](#sandboxmetrics)`>` | Point-in-time resource metrics. |
| start() | `Promise<`[`Sandbox`](#instance-methods)`>` | Resume in attached mode. |
| startDetached() | `Promise<`[`Sandbox`](#instance-methods)`>` | Resume in detached mode. |
| connect() | `Promise<`[`Sandbox`](#instance-methods)`>` | Connect to an already-running sandbox without taking lifecycle ownership. |
| stop() | `Promise<void>` | Graceful shutdown. |
| kill() | `Promise<void>` | Force terminate. |
| remove() | `Promise<void>` | Delete sandbox and all state from disk. |

### LogLevel

Sandbox process log verbosity.

| Value | Description |
|-------|-------------|
| `'trace'` | Most verbose - all diagnostic output |
| `'debug'` | Debug and higher |
| `'info'` | Info and higher |
| `'warn'` | Warnings and errors only |
| `'error'` | Errors only |

### MetricsStream

Async stream returned by [`Sandbox.metricsStream()`](#metricsstream). Supports `Symbol.asyncDispose`.

| Method | Returns | Description |
|--------|---------|-------------|
| [Symbol.asyncIterator] | `AsyncIterator<`[`SandboxMetrics`](#sandboxmetrics)`>` | Use with `for await...of` |
| recv() | `Promise<`[`SandboxMetrics`](#sandboxmetrics)` \| null>` | Receive next snapshot. Returns `null` when the stream ends. |

### PullPolicy

Controls when the SDK fetches an OCI image from the registry.

| Value | Description |
|-------|-------------|
| `'always'` | Pull the image every time, even if cached locally |
| `'if-missing'` | Pull only if the image is not already cached |
| `'never'` | Never pull; fail if the image is not cached locally |

### RootfsSource

Source of the sandbox's root filesystem. Resolved automatically from a string by [`intoRootfsSource(input)`](#rootfssource): a leading `/`, `./`, or `../` becomes a bind path; a `.qcow2` / `.raw` / `.vmdk` extension becomes a disk image; everything else is treated as an OCI reference.

```typescript
type RootfsSource =
  | { kind: "bind"; path: string }
  | { kind: "oci"; reference: string }
  | { kind: "disk"; path: string; format: DiskImageFormat; fstype?: string };

type DiskImageFormat = "qcow2" | "raw" | "vmdk";
```

### SandboxMetrics

Point-in-time resource usage snapshot.

| Field | Type | Description |
|-------|------|-------------|
| cpuPercent | `number` | CPU usage as a percentage |
| memoryBytes | `number` | Current memory usage in bytes |
| memoryLimitBytes | `number` | Memory limit in bytes |
| diskReadBytes | `number` | Total bytes read from disk since boot |
| diskWriteBytes | `number` | Total bytes written to disk since boot |
| netRxBytes | `number` | Total bytes received over the network since boot |
| netTxBytes | `number` | Total bytes sent over the network since boot |
| uptimeMs | `number` | Time since the sandbox was created (ms) |
| timestamp | `Date` | When this measurement was taken |

`allSandboxMetrics()` returns a `Record<string, SandboxMetrics>` keyed by sandbox name.

### SandboxStatus

| Value | Description |
|-------|-------------|
| `'running'` | Guest agent is ready |
| `'stopped'` | VM shut down; can be restarted |
| `'crashed'` | VM exited unexpectedly |
| `'draining'` | Graceful shutdown in progress |

### Sizes

The `microsandbox` package exports type-safe size helpers. All accept a number and return a branded `Mebibytes` value.

| Helper | Description |
|--------|-------------|
| `MiB(n)` | `n` mebibytes. |
| `KiB(n)` | `n` kibibytes (sub-MiB precision). |
| `GiB(n)` | `n` gibibytes. |
| `TiB(n)` | `n` tebibytes. |

Bare numbers are also accepted by APIs taking `Mebibytes | number` (interpreted as MiB).

### Errors

`MicrosandboxError` is the base class. Every Rust-side variant has a dedicated subclass — use `instanceof` instead of parsing message strings.

```typescript
import { Sandbox, SandboxNotFoundError, ExecTimeoutError } from "microsandbox";

try {
  await Sandbox.remove("ghost");
} catch (e) {
  if (e instanceof SandboxNotFoundError) {
    console.log("nothing to remove:", e.message);
  } else {
    throw e;
  }
}
```

Subclasses: `IoError`, `HttpError`, `LibkrunfwNotFoundError`, `DatabaseError`, `InvalidConfigError`, `SandboxNotFoundError`, `SandboxStillRunningError`, `RuntimeError`, `JsonError`, `ProtocolError`, `NixError`, `ExecTimeoutError` (carries `timeoutMs`), `TerminalError`, `SandboxFsError`, `ImageNotFoundError`, `ImageInUseError`, `VolumeNotFoundError`, `VolumeAlreadyExistsError`, `ImageError`, `PatchFailedError`, `CustomError`.
