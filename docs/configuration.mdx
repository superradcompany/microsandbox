microsandbox reads its global configuration from `~/.microsandbox/config.json`. All fields are optional — a missing file or an empty `{}` is equivalent to a fully default configuration.

The config is loaded once on first access. To override values programmatically, call `set_config()` before any other SDK call.

## Full example

```json
{
  "home": "/custom/path/.microsandbox",
  "log_level": "info",
  "database": {
    "url": "sqlite:///tmp/msb.db",
    "max_connections": 10
  },
  "paths": {
    "msb": "/usr/local/bin/msb",
    "libkrunfw": "/usr/local/lib/libkrunfw.so",
    "cache": "/mnt/fast/msb-cache",
    "sandboxes": null,
    "volumes": null,
    "logs": null,
    "secrets": null
  },
  "sandbox_defaults": {
    "cpus": 2,
    "memory_mib": 1024,
    "shell": "/bin/bash",
    "workdir": "/app"
  },
  "registries": {
    "auth": {
      "ghcr.io": {
        "username": "octocat",
        "store": "keyring"
      },
      "registry.example.com": {
        "username": "deploy",
        "password_env": "REGISTRY_TOKEN"
      },
      "docker.io": {
        "username": "user",
        "secret_name": "dockerhub-token"
      }
    }
  }
}
```

## Top-level fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `home` | `string \| null` | `~/.microsandbox` | Root directory for all microsandbox data (sandboxes, volumes, logs, cache, etc.) |
| `log_level` | `string \| null` | `null` (silent) | Default log level for sandbox runtime processes. One of `error`, `warn`, `info`, `debug`, `trace` |
| `database` | `object` | See below | Database connection settings |
| `paths` | `object` | See below | Path overrides for binaries and data directories |
| `sandbox_defaults` | `object` | See below | Default values applied to every sandbox unless overridden |
| `registries` | `object` | See below | Container registry authentication |

## `database`

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `url` | `string \| null` | `null` | Database connection URL. When `null`, uses a SQLite database under the `home` directory |
| `max_connections` | `integer` | `5` | Maximum connection pool size |

## `paths`

All path fields are optional. When `null`, they resolve relative to `home`.

| Field | Default | Description |
|-------|---------|-------------|
| `msb` | `~/.microsandbox/bin/msb` | Path to the `msb` runtime binary. Resolution order: `MSB_PATH` env var → this field → `~/.microsandbox/bin/msb` → `PATH` lookup |
| `libkrunfw` | Sibling to `msb` | Path to the `libkrunfw` shared library (`.so` on Linux, `.dylib` on macOS) |
| `cache` | `~/.microsandbox/cache` | Image layer cache directory |
| `sandboxes` | `~/.microsandbox/sandboxes` | Per-sandbox state directory |
| `volumes` | `~/.microsandbox/volumes` | Named volumes directory |
| `logs` | `~/.microsandbox/logs` | Sandbox log files |
| `secrets` | `~/.microsandbox/secrets` | Secrets directory (registry secret files live under `secrets/registries/`) |

## `sandbox_defaults`

These defaults apply to every sandbox created via the SDK or CLI unless explicitly overridden per-sandbox.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `cpus` | `integer` | `1` | Number of vCPUs |
| `memory_mib` | `integer` | `512` | Guest memory in MiB |
| `shell` | `string` | `"/bin/sh"` | Shell used for interactive sessions and scripts |
| `workdir` | `string \| null` | `null` | Working directory inside the sandbox |

## `registries`

### `registries.auth`

A map of registry hostnames to authentication entries. Each entry specifies a username and exactly **one** credential source.

```json
{
  "registries": {
    "auth": {
      "ghcr.io": {
        "username": "octocat",
        "store": "keyring"
      }
    }
  }
}
```

#### Auth entry fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `username` | `string` | Yes | Registry username |
| `store` | `string \| null` | No | Credential store. Currently only `"keyring"` is supported (OS keyring: Keychain on macOS, Credential Manager on Windows, Secret Service on Linux) |
| `password_env` | `string \| null` | No | Name of an environment variable containing the password or token |
| `secret_name` | `string \| null` | No | Filename under `~/.microsandbox/secrets/registries/` containing the password or token |

<Note>
  Exactly one of `store`, `password_env`, or `secret_name` must be set per entry. Setting none or more than one is an error.
</Note>

### Auth resolution order

When pulling from a registry, credentials are resolved in this order:

1. **Explicit SDK auth** — `.registry_auth()` on the sandbox builder
2. **OS keyring** — entries created by `msb registry login`
3. **Config file** — `registries.auth` entries in `config.json`
4. **Docker config** — `~/.docker/config.json` credential helpers
5. **Anonymous** — no authentication

## Directory layout

When `home` is `~/.microsandbox` (the default), the full directory structure is:

```
~/.microsandbox/
├── config.json
├── bin/
│   └── msb
├── lib/
│   └── libkrunfw.{so,dylib}
├── cache/
├── sandboxes/
├── volumes/
├── logs/
└── secrets/
    └── registries/
```
