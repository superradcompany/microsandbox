External client mode exposes a sandbox as an SSH server. Use it when you want ordinary OpenSSH tooling to connect to a sandbox: `ssh`, `sftp`, editor integrations, or `ProxyCommand`.

Unlike native sessions, external client mode performs public-key authentication. Add public keys explicitly before serving.

## Authorize a key

```bash
msb ssh authorize --file ~/.ssh/id_ed25519.pub
msb ssh authorize --key "ssh-ed25519 AAAA... user@host"
cat ~/.ssh/id_ed25519.pub | msb ssh authorize --stdin
```

| Flag | Description |
|------|-------------|
| `--file PATH` | Read one public key from a file |
| `--key KEY` | Read one public key from the flag value |
| `--stdin` | Read one public key from stdin |

Keys are appended to `<MSB_HOME>/ssh/authorized_keys`. Existing keys are deduplicated by public-key material.

## TCP listener

```bash
msb ssh serve devbox
```

By default, this listens on `127.0.0.1:2222`.

```bash
msb ssh serve devbox --host 127.0.0.1 --port 2223
```

Then connect with ordinary OpenSSH clients:

```bash
ssh -p 2222 root@127.0.0.1
sftp -P 2222 root@127.0.0.1
```

| Flag | Description |
|------|-------------|
| `--host HOST` | Listener host. Defaults to `127.0.0.1` |
| `--port PORT` | Listener port. Defaults to `2222` |

<Note>
  Binding to `0.0.0.0` exposes the SSH listener beyond the local machine. Keep the default loopback bind unless you intentionally want remote clients to connect.
</Note>

## ProxyCommand

`--stdio` serves one SSH transport over stdin/stdout. This is useful when an SSH client should spawn `msb` as its transport bridge instead of connecting to a TCP listener.

```sshconfig
Host devbox.msb
  HostName devbox
  User root
  ProxyCommand msb ssh serve devbox --stdio
```

Now regular OpenSSH commands can use that host alias:

```bash
ssh devbox.msb
sftp devbox.msb
```

## SDK server endpoints

The Rust SDK exposes the most general form: prepare an SSH server endpoint, then pass each ordered duplex stream to `serve(stream)`.

```rust
let server = sandbox
    .ssh()
    .server_with(|ssh| ssh.sftp(true))
    .await?;

server.serve(stream).await?;
```

The TypeScript, Python, and Go SDKs expose server helpers for stdio transports. See the SDK reference pages for the exact language-specific surface.

## SFTP support

SFTP is enabled by default for `msb ssh serve` and SDK server endpoints. It uses the sandbox filesystem protocol, including handle-based reads and writes, directory handles, metadata updates, symlinks, rename, and remove operations.

For direct SDK file access without SSH, use the [Filesystem](/sandboxes/filesystem) API.
